# BugList 文件夹说明

这个文件夹用于记录 GamerTodo 项目中的 Bug，每*个 Bug 单独用一个 Markdown 文件记录*。

## 文件命名规范

- 使用人类可读的名称，例如：`登录按钮点击无响应.md`
- 避免使用数字编号或技术术语
- 尽量描述 Bug 的现象而不是原因

## 文件内容模板

每个 Bug 文件应包含以下信息：

- Bug 标题（文件名）
- 描述：详细的 Bug 现象
- 重现步骤：如何复现这个 Bug
- 预期行为：应该发生什么
- 实际行为：实际发生了什么
- 严重程度：高/中/低
- 状态：新建/已确认/已记录
- 记录时间：YYYY-MM-DD

## 使用说明

- 只记录 Bug，不进行修复
- 可以随时添加新的 Bug 文件
- 文件按时间顺序或严重程度组织
